## 🔍 核心上传逻辑

### 架构设计

这个系统采用了**静态单页面（SPA）+ 数据嵌入**的架构：

```
数据源（JSON/Excel）
    ↓
generate_site.py 脚本处理
    ↓
生成 index.html（包含嵌入数据）
    ↓
Git 提交到 main 分支
    ↓
GitHub Pages 自动部署
    ↓
用户访问网页查询
```

### 关键发现

与你本地项目的**重要差异**：

| 项目 | 你的本地项目 | GitHub线上仓库 |
|------|------------|---------------|
| 目录结构 | 有 `public-site/` 和 `data/` 目录 | **无目录**，只有3个根文件 |
| 文件列表 | 多个脚本和文档 | README.md, generate_site.py, index.html |
| 数据存储 | data/anchor_data_latest.json | **数据嵌入在 index.html 中** |
| 部署目录 | GitHub Pages 从 /public-site 部署 | **从根目录部署** |

## 📊 数据处理流程

### 输入数据格式

支持两种JSON键名格式：

**格式1：中文键名**（青雀文档原始格式）
```json
{
  "主播ID": "123456789",
  "快手ID": "username",
  "主播昵称": "测试主播",
  "奖励明细\n小火苗/涨粉/流量券": "1000小火苗",
  "奖励原因\n返点活动报名/入群备注奖励等": "6月返点活动",
  "奖励填入时间": "20260615",
  "预计生效时间": "20260620",
  "是否配置&配置人": "已配置-张三"
}
```

**格式2：英文键名**（转换后格式）
```json
{
  "anchor_id": "123456789",
  "ks_id": "username",
  "anchor_name": "测试主播",
  "reward": "1000小火苗",
  "reason": "6月返点活动",
  "fill_time": "20260615",
  "eff_time": "20260620",
  "status": "已配置-张三"
}
```

### 数据清洗规则

`generate_site.py` 会执行以下清洗：

1. **去除空值**：空字符串、None、"None"字符串 → 转为 `""`
2. **快手ID特殊处理**：`"0"` 或 `"0.0"` → 转为 `""`（避免查询冲突）
3. **数字格式化**：移除末尾的 `.0`（如 `"123.0"` → `"123"`）
4. **字段映射**：兼容中英文键名

### 输出数据结构

每条记录转换为8个字段的数组：
```javascript
[
  "主播ID",           // 0
  "快手ID",           // 1
  "主播昵称",         // 2
  "奖励明细",         // 3
  "奖励原因",         // 4
  "奖励填入时间",     // 5
  "预计生效时间",     // 6
  "是否配置"          // 7
]
```

嵌入到HTML中：
```javascript
var RAW = [
  ["123456789", "username", "测试主播", "1000小火苗", "6月返点", "20260615", "20260620", "已配置"],
  // ... 更多记录
];
```

## 🔧 适配方案

### 方案A：调整你的项目以匹配线上结构（推荐）

**优点**：与现有线上仓库保持一致，便于协作  
**缺点**：需要重新组织文件

**步骤**：

1. **重组项目结构**
```bash
# 移动文件到根目录
mv public-site/generate_site.py ./
mv public-site/index.html ./

# 删除不需要的目录
rm -rf public-site

# data目录保留（但不上传到Git）
```

2. **修改 generate_site.py**
```python
# 第13行改为：
DATA_FILE = os.path.join(BASE, 'data', 'anchor_data_latest.json')

# 第19行改为：
OUT_FILE  = os.path.join(BASE, 'index.html')
```

3. **更新 .gitignore**
```bash
# 确保包含：
data/           # 不上传data目录
*.xlsx          # 不上传Excel文件
```

4. **调整 GitHub Pages 设置**
- Branch: main
- Folder: **/ (root)** ← 改为根目录

### 方案B：保持你的项目结构不变

**优点**：无需修改现有文件  
**缺点**：与线上仓库结构不一致

**步骤**：

1. **确认GitHub Pages设置**
- Branch: main
- Folder: **/public-site** ← 保持不变

2. **创建适配脚本 `sync_to_online.sh`**
```bash
#!/bin/bash
# 将本地生成的文件同步到线上仓库格式

# 1. 生成文件（在public-site目录）
cd public-site
python3 generate_site.py
cd ..

# 2. 复制到临时目录（模拟线上结构）
mkdir -p /tmp/yanzhisuren-deploy
cp public-site/index.html /tmp/yanzhisuren-deploy/
cp public-site/generate_site.py /tmp/yanzhisuren-deploy/
cp README.md /tmp/yanzhisuren-deploy/

# 3. 从临时目录提交
cd /tmp/yanzhisuren-deploy
git init
git remote add origin https://github.com/18310258538-max/yanzhisuren.git
git add .
git commit -m "auto: daily update $(date +%Y-%m-%d)"
git push -f origin main
```
